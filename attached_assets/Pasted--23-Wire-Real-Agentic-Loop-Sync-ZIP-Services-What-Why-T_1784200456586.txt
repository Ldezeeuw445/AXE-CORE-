#23 - Wire Real Agentic Loop + Sync ZIP Services

What & Why

The ZIP snapshot contains 9 service files (globalMemoryService.ts, aiCoreLogService.ts, tools.ts, agenticEngine.ts, agentMemoryService.ts, aiAgent.ts, ragMemoryService.ts, sharedMemory.ts, geminiLiveService.ts) plus 3 new pages (BrowserPage.tsx, Organization.tsx) and 3 components (ArchitectureCanvas.tsx, ArchitectureRedesign.tsx, OrganizationCanvas.tsx) that exist in the GitHub repo but are missing from the Replit codebase. The LangGraph orchestrator in the ZIP already imports globalMemoryService and aiCoreLogService, so it references stubs that don't exist here yet.

More critically: the three KimiClaw / KimiCode / KimiWork agent panels in AgentChatHub.tsx currently use a getAgentResponse() function that returns randomly-picked hardcoded strings (Math.random()). They are NOT making real LLM calls. The user wants these agents to actually work.

The browser sidebar (AISidebar.tsx) also has a tools.ts tool-calling system in the ZIP (navigate, search, bookmark, summarize_page, etc.) that gives the AI real browser control — this needs to be wired in.

Done looks like

Sending a message to KimiClaw / KimiCode / KimiWork gets a real streaming response from the active LLM provider (same provider pool as the main chat, using each agent's system prompt)
The response shows a small badge indicating which provider answered (e.g. "claude-sonnet-5")
KimiClaw can trigger browser navigation via the tool system (navigate, search) when it recognises a URL/search intent
The AI Core Logs page shows real log entries from orchestration events
All 9 missing service files are present in src/services/, all 3 missing pages are routed in App.tsx, all 3 missing components are importable
The LangGraph orchestrator compiles and health-checks without "module not found" errors
global_memories and ai_core_logs Supabase tables are created (SQL provided to user as plain text to run in dashboard)
Out of scope

Deep EVE skill wiring (separate future task — just ensure EVE page loads without errors)
Ollama / VPS local branch (depends on VPS being reachable; cloud path is what matters here)
Full CrewAI integration (Python backend, separate concern)
Live streaming token-by-token for agent panels (fire-and-forget async call is fine for now)
